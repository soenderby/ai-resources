# Pre-Synthesis Memo

## Is "judgment memory" still the best label?

**Yes, provisionally.** No existing term covers the concept fully:

- "Agent context files" names the container, not the content.
- "Codified context" is broader — it includes factual specs that aren't judgment.
- "Design rationale" is narrower — only decisions, not ongoing preferences or review heuristics.
- "Tribal knowledge" describes the pre-capture state, not the captured artifact.
- "Auto memory" names one capture mechanism, not the concept.

"Judgment memory" remains useful because it names what distinguishes the most valuable entries in agent context files from the least valuable: the presence of human judgment — rationale, heuristics, taste, lessons, and failure-mode knowledge — rather than mere facts or configuration.

The term has a weakness: it implies human origin, but some judgment memory is captured automatically (auto memory, convention detection). A more precise label might be "durable judgment" or "captured judgment," but neither flows as naturally. Keep "judgment memory" as the working label with an explicit note that the production mechanism can be human or automatic.

## What neighboring concepts should remain separate?

The following should **not** be collapsed into judgment memory:

1. **Task memory** — what work exists and its state (Beads, task stores). Different purpose: coordination, not quality.
2. **General context** — project structure, file layout, build commands. Factual, not judgment.
3. **Context engineering** — the design discipline of managing all context. Judgment memory is one kind of content being engineered.
4. **Auto memory (as a mechanism)** — the capture mechanism is distinct from the content. Auto memory produces judgment memory, but also captures non-judgment facts.

The following are **subsets or near-synonyms** that should be acknowledged but not given separate concept entries:

- **Decision memory / design rationale** — the subset about architectural decisions
- **Review memory** — the subset about code review heuristics
- **Captured experience** — judgment memory that originated from failure
- **Team conventions with rationale** — judgment memory encoded as rules
